## 📝 Documentation Format

Each `.md` file follows this structure:
- **Title** — What project I am doing
- **Process** — step-by-step build notes (static; written during development)
- **Errors & Fixes** — issues encountered and how they were resolved
- **New Features** — additions made after initial build
- **Notes** — What I learned in that project
